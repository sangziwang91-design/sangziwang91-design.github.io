Evidence Note

FULL1000 External-Bank BRC Pilot

This page summarizes the public-safe evidence status for a 1,000-record external-bank pilot using BRC-style behavioral coding. It is a pilot evidence note, not a peer-reviewed validation report.

Last structural update: 2026-06-19

Claim ceiling

The current public claim is restricted-access behavioral observability with pilot-level external-bank coding evidence and local runtime validation. This work does not claim direct access to model internals, causal control over model distributions, solved alignment, peer-reviewed reliability, third-party reproducibility, cross-model generalization, or validated civilization-scale governance.

What was run

  • A 1,000-record external-bank pilot was completed with zero run errors.
  • Configured source groups were covered in the run summary.
  • BRC-1 through BRC-8 were observed in item-level coding traces.
  • The run produced calibration issues suitable for follow-up retest.

What is public

  • Public DOI records and public dataset descriptions listed on the main site.
  • Public-safe summary of the FULL1000 pilot and its evidence level.
  • Claim ceiling and next validation gate.
  • High-level disclosure boundary for withheld evidence.

What is supported

The pilot supports a limited statement: externally sourced prompts can be organized into observable BRC-style behavioral coding traces, and the run exposed calibration boundaries that can guide future retest and review.

What is not supported

  • No peer-review outcome is displayed.
  • No current manuscript status is displayed.
  • No claim of production benchmark status is made.
  • No claim of causal control, model-internal measurement, or general population validity is made.

Calibration needs

  • Freeze a public-safe coding manual and rule version.
  • Preserve evidence spans and final-label reasons.
  • Retest boundary cases around BRC-1, BRC-6, and BRC-7.
  • Inspect rare BRC-2, BRC-3, and BRC-5 samples before broader claims.
  • Run a limited blinded scoring check before presenting reliability claims.

Reproducible subset status

A reproducible public subset is not published yet. It should be prepared only after calibration rules, evidence spans, source permissions, and retest outcomes are stable enough for external review.

Disclosure boundary

Internal validation workflows, operational mechanisms, scoring thresholds, intervention heuristics, unpublished logs, and sensitive infrastructure details are withheld. Public materials should be read as a boundary-controlled evidence package, not as a release of internal execution infrastructure.
