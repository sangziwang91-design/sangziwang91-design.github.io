# FULL1000 External-Bank BRC Pilot Note

## 1. What was run

A 1,000-item external-bank pilot run was processed into item-level BRC traces with source coverage and calibration issues visible for follow-up review.

## 2. What was observed

The run produced pilot-level evidence that configured source groups could be processed without run-level errors, and that BRC-1 through BRC-8 labels could be surfaced for review.

## 3. What this supports

This supports a bounded public claim that external-bank prompts can be transformed into reviewable item-level traces with source coverage, label distribution, and calibration questions made visible.

## 4. What this does not prove

This note does not prove BRC coding reliability, third-party reproducibility, cross-model generalization, population-level prevalence, or causal effects on model behavior.

## 5. Calibration issues

Follow-up review is still needed for boundary cases, rare labels, final-label rules, and any case where source coverage or label assignment requires manual calibration.

## 6. Next retest gate

The next gate is to freeze one public-safe BRC rule version, preserve item-level scoring evidence and evidence spans, document final-label rules, retest boundary cases, and run a limited blinded scoring check before broader public claims.

## 7. Claim ceiling

This pilot supports a bounded, pilot-level claim: external-bank prompts can be processed into item-level BRC traces with source coverage and calibration issues visible for follow-up review. It does not prove BRC coding reliability, third-party reproducibility, cross-model generalization, population-level prevalence, or causal effects on model behavior.
